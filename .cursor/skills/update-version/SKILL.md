# Update HARO Version

Bump HARO's release version everywhere it is referenced, refresh `uv.lock`, verify, and commit with the project's naming convention.

This skill does **not** change the Home Assistant minimum version (`hacs.json` + the `homeassistant` constraint in `pyproject.toml`). Those move together as a separate concern.

## Inputs

The target version `X.Y.Z` or `X.Y.ZrcN` (e.g. `0.0.2`, `0.1.0rc1`). If the user did not supply one, ask before continuing.

Version strings in files are bare semver — no leading `v`. The `v` prefix only appears on the git tag.

## Files to update

| File | Field |
|---|---|
| `pyproject.toml` | `version = "..."` under `[project]` |
| `custom_components/haro/manifest.json` | `"version": "..."` |
| `custom_components/haro/const.py` | `VERSION = "..."` |
| `uv.lock` | regenerated by `uv lock` |

The three bare semvers must be identical after the bump.

## Workflow

1. **Confirm target version.** Read the current value in `pyproject.toml`. Refuse to "bump" to the same version unless the user explicitly asks for a re-lock.

2. **Detect drift before editing.** If `manifest.json` or `const.VERSION` already disagrees with `pyproject.toml`, stop and surface the drift. Do not silently align as part of a bump — the agent should ask which version is canonical first.

3. **Update the three version strings** to the new value. Keep the `rc` segment attached (e.g. `0.0.2rc1`, not `0.0.2 rc1`).

4. **Refresh the lockfile:**

   ```bash
   uv lock
   ```

   If the lockfile diff includes anything beyond HARO's own version metadata, surface that to the user before committing.

5. **Verify** (matches CI):

   ```bash
   uv run ruff format --check
   uv run ruff check
   uv run pyright custom_components/haro tests
   uv run pytest
   ```

6. **Commit** with the project's convention (note the `v` prefix on the version in the message, even though the version strings inside files stay bare):

   ```text
   Update version to vX.Y.Z
   ```

   Stage exactly these four paths and nothing else:

   - `pyproject.toml`
   - `custom_components/haro/manifest.json`
   - `custom_components/haro/const.py`
   - `uv.lock`

   If the working tree has unrelated changes, leave them out of this commit.

7. **Push and tag only if the user asks.** When asked to tag, use `git tag vX.Y.Z` (with the `v` prefix) and push the tag explicitly.

## Examples

**Stable bump:** `0.0.1rc7` → `0.0.1`

- `pyproject.toml` → `version = "0.0.1"`
- `manifest.json` → `"version": "0.0.1"`
- `const.py` → `VERSION = "0.0.1"`
- `uv lock`
- commit `Update version to v0.0.1`

**Release candidate:** `0.0.1` → `0.1.0rc1`

- bumps the same three files to `0.1.0rc1`
- commit `Update version to v0.1.0rc1`
- tag (if asked): `v0.1.0rc1`
